Comment React va révolutionner le web

Bibliothèque créée en 2013 par les équipes de Facebook pour gérer l'interface graphique, notamment du web : le DOM et ses API (animations, événements, styles)

Qu'est-ce que ce n'est pas ? React n'est pas un framework. React n'est pas un framework MVC. React n'utilise pas de templates.

React gère les interfaces graphiques grâce à des composants, un peu comme les directives dans Angular. Ceci permet une programmation claire car déclarative plutôt qu'impérative.

React ne s'occupe que de la couche vue. Pour gérer le métier, les équipes de React propose une architecture de single direction data flow : flux de données uni directionnel. Cette architecture réutilise les concepts d'event sourcing pour mettre à jour le modèle.

L'implémentation de flux la plus utilisée est Redux.

D'autres développeurs open source ont créé react-router pour gérer les différentes routes d'un application React.

D'autres développeurs ont créé react-hot-loader, un module pour webpack.

Les équipes de Facebook ont créé react-native après avoir séparé la couche de gestion des composants et la mise à jour du DOM, permettant ainsi de mettre à jour autre chose que le DOM comme les composants graphiques du SDK d'Android ou d'iOS.
